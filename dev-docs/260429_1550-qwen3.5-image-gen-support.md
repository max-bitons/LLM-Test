# Qwen 3.5 影像生成與多模態支援更新

**create by : bitons & cursor**

## 更新內容

我們已修改 `start_vllm_server_qwen3.5.sh` 啟動腳本，以支援影像生成與多模態處理的相關功能：

1. **多模態輸入支援 (Vision Language Models)**
   - 加入了 `--limit-mm-per-prompt image=4,video=1` 參數。
   - 雖然 Qwen 3.5 預設為純文字模型，但若您透過 `QWEN_MODEL_ID` 載入如 `Qwen/Qwen2.5-VL-7B-Instruct` 等多模態模型，此設定能確保 vLLM 伺服器正確處理影像與影片輸入。

2. **工具呼叫 (Tool Calling) 支援**
   - 加入了 `--enable-auto-tool-choice` 與 `--tool-call-parser qwen3_xml` 參數。
   - 由於純文字模型無法直接「生成」影像，透過開啟工具呼叫功能，您可以讓 Qwen 模型在對話中呼叫外部的影像生成 API（如 Stable Diffusion 或 DALL-E）來達成影像生成的目的。

3. **獨立的影像生成 API 伺服器 (可選)**
   - 若您的目標是直接啟動一個 OpenAI 相容的影像生成伺服器（例如給 `run_image_gen_test.py` 測試腳本使用），我們也額外建立了一份 `start_image_server.sh` 腳本。
   - 該腳本預設載入 `stabilityai/stable-diffusion-xl-base-1.0`（需配合支援影像生成的後端如 vllm-omni 或 diffusers API）。

## 檔案變更

- 編輯: `start_vllm_server_qwen3.5.sh`
- 新增: `start_image_server.sh` (提供原生影像生成模型專用啟動腳本參考)
